Aims & overview

This tutorial introduces you to the basics of R and RStudio. You will learn how to - use and knit Markdowns scripts - check & set your working directory - install & load packages - use the help function - understand the concept of R objects & functions - deal with error messages

The working directory

First we check with “getwd()” under which path our working directory is located. This is the folder in which R puts all documents created in this project, and from which it also reads documents. All scripts and data sets we want to work with should be placed in this folder. When we have created a project, this is automatically created as a working directory.

##Check working directory
getwd()
[1] "C:/Users/antje/Documents/studium PUK/Master-Publizistik/STUDIENASSISTENZ/Test Tutorial"

If we want to set a different folder as the working directory, we can best do this manually with the tabs from the top menu: Session -> Set Working Directory -> Choose Directory.

Or we use the “setwd()” function and specify the exact path to the desired folder in the brackets.

Help function

The help function can be used to call the respective help page for all functions, packages, etc. For example, this code opens the help page for the “setwd()” function:

help(setwd)
starte den http Server für die Hilfe fertig

Alternatively, you can click on the “Help” tab in the lower right area and then enter the command in the search field. A window with detailed information will then open.

Install packages

In the following we need two R packages that are not already included in the basic version. An R package is a collection of R functions, data, and documentation developed specifically for a specific purpose or application. R packages are typically built and deployed by the R community. The R packages we need can be downloaded and installed via CRAN (Comprehensive R Archive Network). We can install the packages with the following code:

##Pakete installieren
if (!require(tidyverse)) {install.packages("tidyverse")}

Alternatively, we can click “Packages -> install” at the tabs in the bottom right window. Then another window opens in which we can type the name of the package. Then click “Install” and the package will be installed. When you click “Update”, R Studio will check if there are any updates for installed packages and ask you if it should install them. You only have to install packages once and update them occasionally.

Load packages

Every time we want to use a package in a session for a project, we have to load it first:

##load packages
library(tidyverse)
library(rmarkdown)

The installed packages are listed under “Packages” in the bottom right area. Loaded packages are shown with a tick.

Objects

An object is a variable that stores a value or a collection of values. To create an object in R, you can use the assignment operator “<-”. The left part of the operator gives the name of the object, while the right part contains the value or values to be stored in the object. For example, the following command creates an object “fav_number” and stores the number 9 in it. In the following we can call this object by simply typing its name and also use it for further operations.

##Objects
fav_number <- 9
fav_number
[1] 9
fav_word <- "Zeitgeist"
fav_word
[1] "Zeitgeist"

For example, we can now use our objects in a function.

Functions

With the help of functions we can interact with our objects and do something with them. We call a function by its name. For example there is a function “sqrt” that calculates the square root of an input value. We give the input value in brackets after the function name. When we run this function, R gives us the result as an output.

##Functions
sqrt(9)
sqrt(fav_number)
sqrt(fav_word) #what's wrong?

Data types

When we wanted to use the “sqrt” function with our object “favword”, we got an error message. Why is that? We tried to use a numeric function on a non-numeric object. For every object, a class, or data type is defined which determines what we can do with the object. If we want to find out the data type of “favword” we ask for its “class”:

##Data types
class(fav_word)
[1] "character"
class(fav_number)
[1] "numeric"
is.numeric(fav_word)
[1] FALSE

The following are the most important data types: -integers: e.g., -1,0,1,2,3,…100,… 1000… -doubles/numerics: also fractions and decimal values such as -4.6 and 40.8 -logicals: TRUE or FALSE -characters: “my favorite word”

Now, it’s your turn! Find out whether your favorite number “fav_number” is an integer:

is.integer(fav_number)
[1] FALSE
class(fav_number)
[1] "numeric"
fav_integer <- as.integer(fav_number)
is.integer(fav_integer)
[1] TRUE
