Next, we define the missing/invalid values as “NA”. Remember: In order to identify these values, we check the associated code book! Fortunately, for all eleven variables the missing values are coded in the same way: 7 = refusal, 8 = don’t know, 9 = no answer. Thus, we can recode all values in the data with just one line of code.

To do this, we use the mutate_all() function from the dplyr package, which is part of the Tidyverse. The “~ifelse(. %in% c(7, 8, 9), NA, .)” statement checks whether each value in the variables is either 7, 8, or 9. If so, the value is replaced with NA; otherwise, the original value is retained. Alternatively, we can proceed as in Exercise 5 and apply the “Recode” function to each variable separately. This takes a bit of work, but it works the same way.

#Recode missing values
df1 <- df1 %>%
  mutate_all(~ifelse(. %in% c(7, 8, 9), NA, .))

We check with View() if it worked.

View(df1)

For further analysis, we exclude the cases with missing values using the “drop_na” function. We apply it to the entire data set because we exclude all cases with missing values at once. With View() we can look at the filtered data set and check if it worked.

In our case, the correlation matrix shows consistently weak to moderate correlations ( 0.1 < r < 0.5). Using the Bartlett test, we check whether the correlation matrix deviates significantly from an identity matrix in which all correlations are 0. To do this, we apply the “cortest.bartlett” function to our R matrix. Here, we also have to specify the size of the sample, i.e., the number of cases in our data set, as an argument. Alternatively, we can also apply the function directly to our data set “df1”. We can ignore the warning that R now gives us.

In addition, we calculate the Kaiser-Meyer-Olkin (KMO) measures of sampling adequacy (MSA) for the variables. They indicate whether there is enough data for each factor to obtain reliable results for factor analysis. The values should be at least > 0.5.

Factor extraction

We first perform a principal component analysis (PCA) without rotation to make a decision on how many factors (components) we want to extract. We use the “principal” function from the “psych” package and set the number of factors (nfactors) equal to the number of variables (=length of the data vector; length(df1)) and the rotation method (rotate) to “none”. We use the correlation matrix “R_df1” as a basis. However, you can also use the “df1” data set directly.

We also use the “plot” function to create a scree plot in which we plot the eigenvalues for the factors. With “abline(h=1)” we add a horizontal line at value 1 to better check the Kaiser criterion (eigenvalue h > 1).

  # Unrotated PCA
pc1 <- principal(R_df1, nfactors = length(df1), rotate="none")

First we see the R output for our call. The following values are returned for each extracted component: - SS loadings: These are the eigenvalues (sum of squared factor loadings) of the factors. - Proportion Var: If you divide the eigenvalue by the number of components, you get the share of the variance explained by the factor in relation to the total variance. For example, PC1 explains 32% of the total variance. - Cumulative Var: The explained variance is summed up over all components. - Proportion Explained: sets the variance explained by the components in relation to the total explained variance, i.e. here the explained variance always adds up to 1, while Proportion Var only adds up to 1 if the entire variance in the data set is explained by the components. Because we extracted the maximum number of factors here, Proportion Var and Proportion Explained are the same. - Cumulative Proportion: The Proportion Explained is summed up over all components.

Next, we see a table (data.frame) that contains the correlations between variables and components (factor loadings) in the first columns. Since these are still based on the unrotated matrix, they are not meaningful and we do not interpret them further. This is followed by a column (h2) showing the communalities, i.e. the variance of a variable explained by all factors together. Since we extracted as many factors as variables in our first PCA, the communalities for all variables are 1. The following column contains values for the “uniqueness” (u2), i.e. the unexplained part of the variance of a variable (h2-1). This is followed by the “com” column, which lists a measure of the complexity of the variable. Complexity represents the number of components required to explain the observed variable. A perfect solution with a simple structure would have a complexity of 1 since each item would only load on one factor.

We can see from the scree plot and the table with the eigenvalues (SS loadings) that the third component still has an eigenvalue greater than 1, but all the others are below it. Therefore, for further analysis, we extract three factors.

We now repeat the PCA and enter “nfactors = 3” and the rotation method “rotate = oblimin”, i.e. we rotate in such a way that variables only load on one factor if possible and allow the factors to correlate with each other.

The output is the same as the previous one, but now only three components are extracted and accordingly we only get information for these three components. In the “Proportion Var” row, we see that the first factor explains 19% of the total variance, the second factor 18%, and the third 16%. Together, all three factors explain 53% of the total variance in the variables (see the “Cumulative Var” row, which sums up the explained variance from factor to factor).

A table with the correlations between the components is now also displayed in the R Console. We see that all three factors correlate moderately with each other.

Now we can clearly see which items load on which component. Factor 1 summarizes the items cptppola (confidence in one’s own ability to participate), actrolga (activity in a political group) and polintr (political interest). This could be interpreted as “belief in political effectiveness”. The items pstplonl (online posts), sgnptit (petitions), pbldmn (demonstrations), bctprd (boycott) load at factor 2. These all involve more or less symbolic political activities. Factor 3 also includes the items wrkprty (work for a political party) and contplt (contacts with politics), and one could describe it as more practical political activities.
