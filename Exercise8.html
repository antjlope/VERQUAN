Aims

In this tutorial you will learn how to check for and improve the quality and reliability of factor solutions. Specifically, you will learn how to:

  • calculate and further use factor scores
  • check and improve interpretability of factor solutions
  • check reliability of the factors as measurement scales of a larger construct

Reliability analysis of the factors

In the pattern matrix we can see which items load high on a factor. We can think of the items as a scale, with which we measure a common construct (e.g., symbolic political activities). By calculating Cronbach’s Alpha below, we obtain a measure of the internal consistency of a factor: how strongly do the individual items on a scale correlate with one another, or in other words: how well do they measure the overall construct together?

We are pleased that our three factors now explain a little more variance at 56%.

Based on the new pattern matrix, we define the factors with their items as measurement scales. To do this, we note in which column our variable appears in the data frame.

  1. Political self-efficacy: cptppola (3), actrolga (2), polintr (1)
  2. Symbolic political activity: bctprd (9), sgnptit (7), pstplonl (10), pbldmn (8)
  3. Practical political activity: wrkprty (5), contplt (4), wrkorg (6)

We store the items in their own objects that correspond to the factors. Here we specify the vector “c” with the indexed position of the variable in the dataframe.

  # Define scales
politEff <-df2[, c(1, 2, 3)]
symbAct <- df2[, c(7, 8, 9, 10)]
prAct <- df2[, c(4, 5, 6)]

Now we calculate Cronbach’s Alpha using the “alpha” function from the “psych” package.

But stop! We remember that the item “polintr” was coded inversely, i.e. the other way around than the other items. Since this can skew the results of the reliability analysis, it is imperative that we tell R to treat this item inversely. To do this, we use the “keys” argument and give R a list of values of 1 or -1 for each item: 1 = item positively coded and -1 = item negatively (i.e. inversely) coded. If all values are positive, i.e. equal to 1, we don’t need to set “keys” as that is the default. So we only need the argument for the first factor “politWirk”.

Alternatively, you can also recode the item beforehand with “Recode”.

 # Cronbach's Alpha
alpha(politEff, keys = c(-1, 1, 1))

We get four tables as output for each scale/factor. We’ll just take a closer look at the first and third tables.

Table 1: Here we are interested in the value for “raw alpha”. This is the value for Cronbach’s Alpha. We ignore the other values.

Table 3: Contains the Cronbach’s alpha value (“raw_alpha”) for each item, if this item is excluded. If the scale becomes much more reliable if an item is excluded, it is advisable to repeat the factor analysis and reliability analysis without this item.

Factors are reliable when Cronbach’s alpha > 0.7

This is the case for factor 1, but not for the other two factors. Only factor 1, political self-efficacy, passes our reliability test. Unfortunately, with factors 2 and 3, excluding individual items does not help either. Cronbach’s alpha does not rise above 0.7 as a result.
